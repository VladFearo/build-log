# build-log

Small finished programming projects, tracked over time.

## Purpose

This repo is a running log of small builds and practice projects.
Each folder is a closed problem with a clear goal, a finished solution, and a short README.

The point is not to build one giant system.
The point is to leave behind proof of work over time.

## Structure

Each project lives in its own folder, usually named like:

`YYYY-MM-DD-project-name`

Example:

```text id="4z5vyt"
projects/
  2026-04-18-summarize-numbers/
    README.md
    summarize-numbers.js
```

## Rules

- one folder = one finished project
- keep scope small
- prefer finished over ambitious
- avoid endless tinkering
- if something is revisited later, treat it as a new version or separate project

## Standard project contents

Each project should usually include:

- `README.md`
- one main code file
- optional sample input/output file if needed

## What each project README should contain

- date
- goal
- what it does
- main features
- example input/output if useful
- what was practiced
- file name

## Why this exists

This repo acts as:

- a diary of consistent work
- a record of solved problems
- a low-friction way to practice
- a visible history of progress

## Philosophy

Small completed builds compound better than large unfinished ones.
This repo is meant to favor clarity, closure, and consistency.
